# Go元编程示例

下面给出三个从入门到进阶的完整可运行示例，覆盖 Go 元编程最常用的三种形态，每段代码都可以直接复制运行。

### 示例一：反射解析结构体标签（运行时元编程）
这是最基础的元语言应用：**结构体标签是元语言，描述字段的序列化规则；Go 结构体本身是对象语言；`reflect` 包是元操作接口**，负责读取元数据并驱动业务逻辑。

下面实现一个极简的 JSON 标签解析器，模拟标准库 `encoding/json` 的核心逻辑。

```go
package main

import (
	"fmt"
	"reflect"
	"strings"
)

// 对象语言：业务结构体
type User struct {
	ID   int    `json:"user_id"`
	Name string `json:"user_name,omitempty"`
	Age  int    `json:"-"` // - 表示忽略该字段
}

// 元逻辑：通过反射读取元数据（标签），驱动序列化行为
func parseStructTags(obj interface{}) {
	t := reflect.TypeOf(obj)
	v := reflect.ValueOf(obj)

	// 只处理结构体类型
	if t.Kind() != reflect.Struct {
		fmt.Println("only support struct")
		return
	}

	fmt.Printf("解析结构体: %s\n", t.Name())
	for i := 0; i < t.NumField(); i++ {
		field := t.Field(i)
		value := v.Field(i)

		// 读取元语言（结构体标签）
		tagStr := field.Tag.Get("json")
		if tagStr == "" {
			fmt.Printf("  字段 %s: 无json标签，使用默认字段名\n", field.Name)
			continue
		}

		// 解析标签规则
		tagParts := strings.Split(tagStr, ",")
		jsonFieldName := tagParts[0]
		omitEmpty := len(tagParts) > 1 && tagParts[1] == "omitempty"

		if jsonFieldName == "-" {
			fmt.Printf("  字段 %s: 标记为忽略，不参与序列化\n", field.Name)
			continue
		}

		fmt.Printf("  字段 %s -> JSON键名: %s, 忽略空值: %v, 当前值: %v\n",
			field.Name, jsonFieldName, omitEmpty, value.Interface())
	}
}

func main() {
	user := User{ID: 1001, Name: "", Age: 25}
	parseStructTags(user)
}
```

**运行结果**：
```
解析结构体: User
  字段 ID -> JSON键名: user_id, 忽略空值: false, 当前值: 1001
  字段 Name -> JSON键名: user_name, 忽略空值: true, 当前值: 
  字段 Age: 标记为忽略，不参与序列化
```

**原理解析**：标签本身只是字符串，不具备任何执行能力。通过反射（元层接口）读取标签中的元规则，再由业务逻辑解释执行，就实现了「元描述驱动对象行为」的元语言模式。

### 示例二：`go generate` + 模板生成 CRUD 代码（编译前代码生成）
这是 Go 官方最推荐的元编程范式：**在编译前用元程序批量生成目标代码，生成结果完全可见、可调试、零运行时开销**。这里 `text/template` 模板语法是元语言，最终生成的 DAO 层代码是对象语言。

#### 文件结构
```
.
├── model
│   └── user.go       // 业务模型 + generate 指令
└── cmd
    └── gen
        └── main.go   // 代码生成器（元程序）
```

#### 1. 业务模型文件 `model/user.go`
```go
package model

//go:generate go run ../cmd/gen

// User 用户模型（对象语言：被描述的业务实体）
type User struct {
	ID   int    `json:"user_id"`
	Name string `json:"user_name"`
	Age  int    `json:"age"`
}
```

#### 2. 代码生成器 `cmd/gen/main.go`
```go
package main

import (
	"os"
	"text/template"
)

// 元数据：描述要生成代码的结构体信息
type Field struct {
	Name string // 字段名
	Type string // 字段类型
}

type ModelInfo struct {
	StructName string  // 结构体名
	TableName  string  // 对应表名
	Fields     []Field // 字段列表
}

// 元语言：CRUD 代码模板
const daoTemplate = `// Code generated by gen; DO NOT EDIT.
package model

import "fmt"

// {{.StructName}}DAO 自动生成的数据访问对象
type {{.StructName}}DAO struct{}

// New{{.StructName}}DAO 构造函数
func New{{.StructName}}DAO() *{{.StructName}}DAO {
	return &{{.StructName}}DAO{}
}

// Insert 插入记录
func (dao *{{.StructName}}DAO) Insert(m *{{.StructName}}) error {
	fmt.Println("插入表 {{.TableName}}，ID:", m.ID)
	// 实际项目中这里拼接 SQL 或调用 ORM
	return nil
}

// GetByID 根据ID查询
func (dao *{{.StructName}}DAO) GetByID(id int) (*{{.StructName}}, error) {
	fmt.Println("查询表 {{.TableName}}，ID:", id)
	return &{{.StructName}}{ID: id}, nil
}

// Update 更新记录
func (dao *{{.StructName}}DAO) Update(m *{{.StructName}}) error {
	fmt.Println("更新表 {{.TableName}}，ID:", m.ID)
	return nil
}

// Delete 删除记录
func (dao *{{.StructName}}DAO) Delete(id int) error {
	fmt.Println("删除表 {{.TableName}}，ID:", id)
	return nil
}
`

func main() {
	// 元数据：定义要生成的模型信息（实际项目中可通过 AST 解析源码自动提取）
	userModel := ModelInfo{
		StructName: "User",
		TableName:  "t_user",
		Fields: []Field{
			{Name: "ID", Type: "int"},
			{Name: "Name", Type: "string"},
			{Name: "Age", Type: "int"},
		},
	}

	// 解析模板
	tpl, err := template.New("dao").Parse(daoTemplate)
	if err != nil {
		panic(err)
	}

	// 生成代码文件
	outFile, err := os.Create("model/user_dao.go")
	if err != nil {
		panic(err)
	}
	defer outFile.Close()

	err = tpl.Execute(outFile, userModel)
	if err != nil {
		panic(err)
	}

	println("已生成 model/user_dao.go")
}
```

#### 运行方式
在项目根目录执行：
```bash
go generate ./model/
```

执行后会自动生成 `model/user_dao.go` 文件，包含完整的 CRUD 方法。工业级项目中，通常会用 AST 解析 `user.go` 自动提取字段信息，无需手动维护元数据。

### 示例三：AST 遍历自动补全 JSON 标签（源码级元操作）
这是更硬核的元语言应用：**用 Go 标准库操作源代码的 AST（抽象语法树），批量修改源码**。这里 AST 操作接口是元语言，被修改的 Go 源文件是对象语言，是实现代码检查、重构工具的基础。

下面实现一个小工具：自动给 Go 结构体字段补上蛇形命名的 `json` 标签，已存在标签的字段跳过。

```go
package main

import (
	"bytes"
	"fmt"
	"go/ast"
	"go/format"
	"go/parser"
	"go/token"
	"strings"
	"unicode"
)

// 驼峰转蛇形命名
func camelToSnake(s string) string {
	var buf bytes.Buffer
	for i, r := range s {
		if unicode.IsUpper(r) {
			if i > 0 {
				buf.WriteByte('_')
			}
			buf.WriteRune(unicode.ToLower(r))
		} else {
			buf.WriteRune(r)
		}
	}
	return buf.String()
}

func main() {
	// 待处理的原始源码（对象语言）
	src := `package demo

type User struct {
	ID    int
	Name  string
	Age   int
	Email string ` + "`gorm:\"column:email\"`" + `
}
`

	fset := token.NewFileSet()
	// 1. 将源码解析为 AST（元层抽象表示）
	f, err := parser.ParseFile(fset, "demo.go", src, parser.ParseComments)
	if err != nil {
		panic(err)
	}

	// 2. 遍历 AST，定位结构体并修改标签（元层操作）
	ast.Inspect(f, func(n ast.Node) bool {
		structType, ok := n.(*ast.StructType)
		if !ok {
			return true
		}

		for _, field := range structType.Fields.List {
			// 已存在 json 标签则跳过
			if field.Tag != nil && strings.Contains(field.Tag.Value, "json:") {
				continue
			}
			if len(field.Names) == 0 {
				continue
			}

			fieldName := field.Names[0].Name
			jsonTag := camelToSnake(fieldName)
			newTagContent := fmt.Sprintf(`json:"%s"`, jsonTag)

			if field.Tag == nil {
				field.Tag = &ast.BasicLit{
					Kind:  token.STRING,
					Value: "`" + newTagContent + "`",
				}
			} else {
				// 追加到原有标签后
				oldTag := strings.Trim(field.Tag.Value, "`")
				field.Tag.Value = "`" + oldTag + " " + newTagContent + "`"
			}
		}
		return true
	})

	// 3. 将修改后的 AST 还原为格式化源码
	var buf bytes.Buffer
	if err := format.Node(&buf, fset, f); err != nil {
		panic(err)
	}

	fmt.Println("===== 自动补全标签后的源码 =====")
	fmt.Println(buf.String())
}
```

**运行结果**：
```go
package demo

type User struct {
	ID    int    `json:"id"`
	Name  string `json:"name"`
	Age   int    `json:"age"`
	Email string `gorm:"column:email" json:"email"`
}
```

**原理解析**：整个过程是「源码 → AST（元层表示）→ 修改AST → 还原源码」，我们在元层级面对对象语言（源代码）进行结构化操作，比纯文本替换更精准、更可靠。
